Pliki konfiguracyjne zostały stworzone!

Następne kroki:

  • Uzupełnij plik o nazwię "attracions_nazwamiasta.json" atrakcjami według wzoru podanego w pliku. Każda atrakcja powinna posiadać następujące atrybuty:
    • "category" -> kategorie danej atrakcji, lista z kategoriami. Dostępne kategorie:
      • Architektura
      • Koscioly i katedry
      • Muzea
      • Parki i ogrody
      • Punkt widokowy
      • Rekreacja
    • "popularity" -> popularność danej atrakcji - liczba recenzji z google
    • "rating" -> ocena danej atrakcji - średnia ocena recenzji z google
    • "price" -> cena atrakcji w euro
    • "timespend" -> średni czas zwiedzania atrakcji w minutach
    • "openinghours" -> godziny otwarcia. Lista z dwoma godzinami. Jeżeli atrakcja jest otwarta całą dobę: lista z jednym elementem -> ["all day"]
    • "cords" -> słownik z koordynatami
  • Wywołanie funkcji utils.py w celu obliczenia odległości między atrakcjami. Jako parametr funkcji podajemy nazwę miasta. Jedno wywołanie funckji oblicza odległośći dla jednej atrakcji (ze względu na ograniczoną ilość zapytań API serwisu openrouteservice)

Przykład pliku z atrakcjami:

            {
                "attracion_name1": {
                    "category": [
                        "Architektura",
                        "Muzea"
                    ],
                    "popularity": 0,
                    "rating": 0,
                    "price": 0,
                    "timespend": 30,
                    "openinghours": [
                        "9:00",
                        "19:00"
                    ],
                    "cords": {
                        "latitude": 42.123123,
                        "longitude": 12.32323
                    }
                },
            }
